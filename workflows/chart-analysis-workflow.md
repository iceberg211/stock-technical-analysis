# 图表分析工作流 (Chart Analysis Workflow)

> **当用户发送一张或多张（同品种多周期）K 线图，或要求使用 MCP / OHLC 数据分析同一品种时，AI 必须按以下流程逐步分析，不可跳步。**

## 执行原则

1. **先分析，后结论**：必须先完成 Step 0~7，再输出任何方向、点位、交易方案。
2. **主动获取数据**：如果用户要求分析某标的但未提供任何图表或数据，AI **必须优先**利用可用的 MCP Server 搜索或拉取该标的的最新 OHLC 价格数据，待获取后再执行 Step 0。若无法获取，须告知用户补充图表。
3. **数据优先**：信息可信度优先级为 `MCP / OHLC 数据 > 用户明确给出的数值 > 清晰截图 > 模糊截图`。
4. **最小读取原则**：默认使用本 workflow 内置规则，不要为每一步机械打开 `references/`；只有遇到边界情况时才读取单个相关文件。
5. **图像保守原则**：只有截图时，若价格刻度或最近 3~5 根 K 线细节不清晰，只做结构和区域判断，不做微观 K 线强判断。

---

## 分析流程

```
收到输入（K线图 / MCP-OHLC数据 / 二者皆有）
  │
  ├── Step 0: 分析模式识别
  ├── Step 1: 基础信息识别
  ├── Step 1.5: 多时间框架映射（仅多图时）
  ├── Step 2: 市场结构判断
  ├── Step 3: 关键支撑阻力标记
  ├── Step 4: 当前价格行为分析
  ├── Step 5: 形态识别
  ├── Step 6: 指标信号（如图上可见）
  ├── Step 7: 综合研判
  └── Step 8: 交易方案（如有信号）
```

---

## Step 0: 分析模式识别

先识别本次分析的数据来源，并据此决定能否给出具体价位与微观判断。

| 模式 | 数据来源 | 允许输出 |
|------|---------|---------|
| **数据模式** | MCP / OHLC / API 返回的结构化行情 | 可做结构分析、关键位分析、最近 K 线分析、具体条件价位 |
| **图片模式** | 用户仅提供截图 | 以结构、区域、明显形态为主；价格刻度不清时禁止输出具体价位 |
| **混合模式** | 同时有截图和结构化数据 | 以数据为准，截图用于辅助交叉验证 |

输出：`分析模式: [数据模式 / 图片模式 / 混合模式]`

---

## Step 1: 基础信息识别

**先看清楚图的基本信息，不要急于判断方向。**

| 识别项 | 说明 |
|--------|------|
| 品种 | 什么资产？（股票/期货/加密货币） |
| 图表数量 | 一张还是多张？是否为同一交易品种？ |
| 时间框架 | 什么级别？（1分钟/15分钟/1小时/4小时/日线/周线） |
| 可见K线数量 | 大约能看到多少根K线？ |
| 数据来源 | 截图 / MCP / OHLC / 混合 |
| 图上指标 | 有没有叠加 RSI/MACD/均线/布林带等？ |
| 当前价格 | 图表最右侧的当前价格大约在什么位置？ |

> 如果图上信息不清楚，**主动向用户确认**，不要猜测。

### 市场类型适配

根据 Step 1 识别的品种，在后续分析中自动适配：

| 市场类型 | 适配要点 |
|---------|---------|
| 股票 | 注意开盘跳空、盘前盘后、财报日历、交易时间限制 |
| 期货 | 注意合约到期、交割月切换、保证金变动 |
| 加密货币 | 24/7 交易无跳空、波动率更高、资金费率影响、无涨跌停限制 |
| 外汇 | 交易时段（亚/欧/美）影响波动性、注意利率决议 |

> AI 不需要在输出中专门写一段"市场适配说明"，而是在后续 Step 2-8 中自然融合上述要点。

### 硬性门槛（不满足则停止）

| 条件 | 行动 |
|------|------|
| 无法识别品种或时间框架 | ❌ 停止分析，向用户确认 |
| 多张图但品种不一致 | ❌ 停止合并分析，要求用户按品种拆分 |
| 价格刻度不可读/模糊 | ⚠️ 可继续到 Step 7 给出方向判断，但 **Step 8 禁止输出具体价格**，只给条件描述 |
| 只有截图且最近 3~5 根 K 线细节不清晰 | ⚠️ Step 4 只能输出“粗略动量判断”，不能输出长影线/吞没/衰竭等强结论 |
| 图片严重模糊/截断 | ❌ 停止分析，请用户重新截图 |

## Step 1.5: 多时间框架映射（仅多图时）

**仅在同一品种上传了 2 张及以上不同周期图时执行。**

### 1.5.1 角色分配（先大后小）

- 将最高周期图定义为 **高周期（HTF）**：用于定方向与关键区间
- 将较低周期图定义为 **低周期（LTF）**：用于找触发与风控细节

### 1.5.2 常见周期组合（示例）

| HTF（定方向） | LTF（找时机） |
|--------------|---------------|
| 日线 (1D) | 4小时 (4H) / 1小时 (1H) |
| 4小时 (4H) | 1小时 (1H) / 15分钟 (15m) |
| 1小时 (1H) | 15分钟 (15m) / 5分钟 (5m) |

### 1.5.3 一致性检查（硬性）

| 场景 | 处理规则 |
|------|---------|
| HTF 偏多，LTF 出现看多触发 | ✅ 允许给做多方案 |
| HTF 偏空，LTF 出现看空触发 | ✅ 允许给做空方案 |
| HTF 与 LTF 方向冲突 | ❌ 不给具体方案，先观望等待 LTF 回归 HTF 方向 |
| HTF 震荡/混乱 | ⚠️ 降低信心等级，优先等待突破后再评估 |

**输出**：`多周期映射: [HTF=xxx, LTF=xxx] / 一致性: [一致 / 冲突 / 中性]`

---

## Step 2: 市场结构判断

**这是最重要的一步。先定方向，再看信号。**

默认按本节内置规则执行；只有在用户明确追问 BOS / CHoCH 定义或结构争议较大时，才读取 `references/core/market-structure.md`。

### 2.1 标记主要 Swing 高低点

- 找到图表中最明显的 **Swing High** 和 **Swing Low**
- 只标记 Major Swing（一眼能看到的），忽略 Minor Swing

### 2.2 判断市场状态

| 状态 | 特征 | 结论 |
|------|------|------|
| **上升趋势** | Higher Highs + Higher Lows | 只找做多机会 |
| **下降趋势** | Lower Highs + Lower Lows | 只找做空机会 |
| **震荡区间** | 高低点大致走平 | 区间高抛低吸 或 等待突破 |
| **混乱/过渡** | 无明显规律 | ❌ 不建议交易 |

### 2.3 检查 BOS / CHoCH

- 最近是否有 BOS（趋势延续确认）？
- 最近是否有 CHoCH（趋势可能变化的预警）？
- 如有 CHoCH，是否已有反向 BOS 确认反转？

### 2.4 评估趋势健康度

- Impulse（推动浪）：实体大、方向一致、速度快？
- Pullback（回调浪）：实体小、影线多、颜色混杂？
- 回调是否健康？（Fib 0.382~0.618 区间内？）

**输出**：`市场状态: [上升趋势 / 下降趋势 / 震荡 / 混乱]`

---

## Step 3: 关键支撑阻力标记

默认按本节内置规则执行；只有在需要扩展到供需区、翻转位细分时，才读取 `references/core/support-resistance.md`。

### 3.1 找最明显的水平

- 缩小看全局：有哪些价格区域被多次触碰？
- 只标记**最明显的 2~3 个**，不要画满屏幕

### 3.2 判断当前价格位置

| 位置 | 含义 |
|------|------|
| 靠近支撑区域 | 关注反弹信号 或 跌破风险 |
| 靠近阻力区域 | 关注突破信号 或 回落风险 |
| 在中间位置 | "无人区"，信号模糊，等待靠近关键位 |

### 3.3 检查翻转

- 有没有曾经的支撑被跌破后变成了阻力？
- 有没有曾经的阻力被突破后变成了支撑？

**输出**：`当前位置: [在支撑附近 / 在阻力附近 / 中间无人区]`

---

## Step 4: 当前价格行为分析

默认按本节内置规则执行；只有在需要细分真假突破、复杂价格行为特征时，才读取 `references/core/price-action.md`。

如果是**图片模式**且最近 3~5 根 K 线不够清晰：

- 只输出：`多头主导 / 空头主导 / 犹豫不决 / 无法细判`
- 不要输出：`长上影拒绝 / 长下影拒绝 / 动量蜡烛 / 衰竭吞没` 这类依赖细节像素的强判断

### 4.1 实体分析

- 实体在变大还是变小？（动量增强 or 衰竭？）
- 多方还是空方占主导？

### 4.2 影线分析

- 有没有长影线出现在关键位？（被拒绝的信号）
- 上影线多（上方有卖压）还是下影线多（下方有买压）？

### 4.3 动量判断

- 有没有出现动量蜡烛（实体 ≥ 前3根的2倍）？
- 动量方向与趋势方向是否一致？

**输出**：`近期行为: [多头主导 / 空头主导 / 犹豫不决 / 动量衰竭 / 无法细判]`

---

## Step 5: 形态识别

默认只识别高频、低歧义形态；不要为了“找形态”而机械打开全部 pattern references。

### 5.1 K线形态检查

- 最近 1~5 根 K线是否构成已知的反转/延续形态？
- 默认只识别：锤子线、射击之星、吞没、启明星、黄昏星
- 如果截图无法清晰区分锤子 / 上吊、射击之星 / 倒锤子，直接写 `无明显形态`
- 只有用户明确追问某个复杂 K 线形态时，才读取 `references/patterns/candlestick-patterns.md`

### 5.2 图表形态检查

- 更大范围内是否在形成图表形态（双底、头肩、旗形、三角形等）？
- 形态是否已完成？还是正在形成中？
- 默认只识别：双顶/双底、头肩/倒头肩、旗形、三角形
- 只有用户明确追问复杂图表形态时，才读取 `references/patterns/chart-patterns.md`

### 5.3 信号强度评估

如果识别到形态，按对应形态章节中的**信号强度评分**打分：

| 评级 | 条件 |
|------|------|
| **强信号** (≥4分) | 形态 + 关键位 + 趋势方向 + 指标/量能确认 |
| **中等信号** (2~3分) | 形态 + 关键位 或 形态 + 趋势方向 |
| **弱信号** (≤1分) | 仅有形态，无其他确认 → 建议观望 |

**输出**：`识别形态: [形态名称] / 信号强度: [弱/中/强] / 无明显形态`

---

## Step 6: 指标信号（如图上可见）

默认只做轻量指标判断，不要为了补全流程而读取全部指标资料。

如果图表上叠加了指标，或结构化数据中已提供可直接判断的指标，再分析：

### RSI（如可见）
- 当前 RSI 位置（超买/超卖/中性）
- 是否存在背离？
- RSI 与趋势方向是否一致？

### MACD（如可见）
- MACD 线位置（零轴上方/下方）
- 是否存在背离？
- 柱状图是否在扩大/缩小？

### 其他指标
- 均线排列方向？
- 布林带收缩/扩张？

> 如果图上没有指标、数据中也未提供指标，跳过此步骤，不要凭空猜测。
> 只有用户明确要求 RSI / MACD / SMC 的详细解释时，才读取 `references/indicators/` 下对应文件。

**输出**：`指标确认: [确认趋势 / 出现背离 / 无指标可判断]`

---

## Step 7: 综合研判

将 Step 2~6 的结果汇总，给出综合判断：

> 本步骤先产出“分析结论”，不要抢先输出交易决策卡或具体价位。

### 多空偏向

| 条件组合 | 偏向 |
|---------|------|
| 上升结构 + 支撑附近 + 看涨形态 | 🟢 偏多 |
| 下降结构 + 阻力附近 + 看跌形态 | 🔴 偏空 |
| 震荡 + 关键位 + 反转形态 | 🟡 区间交易 |
| 结构不清 / 无明确形态 / 中间位置 | ⚪ 观望 |

### 信心等级

| 等级 | 条件 |
|------|------|
| **高信心** | 结构清晰 + 关键位 + 形态确认 + 指标确认（4重汇合） |
| **中等信心** | 3 个因素汇合 |
| **低信心** | 仅 1~2 个因素 → 建议观望或等待更多确认 |

### 多时间框架一致性（多图时必做）

| HTF 结论 | LTF 触发 | 最终偏向 |
|---------|---------|---------|
| 偏多 | 看多触发 | 🟢 偏多（可执行） |
| 偏空 | 看空触发 | 🔴 偏空（可执行） |
| 偏多 | 看空触发 | ⚪ 冲突，观望 |
| 偏空 | 看多触发 | ⚪ 冲突，观望 |
| 震荡/混乱 | 任意触发 | 🟡 降级处理，谨慎 |

---

## Step 8: 交易方案（如有信号）

**只有在信号强度为"中"或"强"时才给出具体方案。弱信号只给观察建议。**

> **强制自查 (Sanity Check)**：在输出方案与计算仓位前，务必默默自查：1) 信号方向是否与大周期趋势冲突？2) 是否存在事件冲突等硬性否决项？

### 8.1 前置检查（硬性门槛）

| 条件 | 行动 |
|------|------|
| 价格刻度不可读 | ❌ **禁止输出具体入场价/止损价/目标价**，改为描述条件（如"突破前高时入场""形态最低点下方止损"） |
| 信号强度为"弱" | ❌ 不输出交易方案，只给观察建议 |
| 市场状态为"混乱" | ❌ 不输出交易方案，建议观望 |
| 多周期一致性为"冲突" | ❌ 不输出交易方案，等待低周期重新与高周期同向 |

### 8.2 Playbook 匹配

默认先按下表快速匹配，不必先打开 `references/playbooks/INDEX.md`：

| Setup | 最少条件 | 行动 |
|------|---------|------|
| `trend-pullback` | 趋势清晰（至少 2 次同向 BOS）+ 无反向 CHoCH + 正在回调到关键位 + 出现反转信号 | 可准备顺势方案 |
| `breakout-retest` | 关键位被动量实体突破 + 收盘站上/跌破 + 回踩突破位附近 + 出现拒绝 PA | 可准备突破回踩方案 |
| `range-reversal` | 成熟震荡区间 + 价格靠近边缘 20% + 出现反转/拒绝信号 | 可准备区间反转方案 |
| `无匹配` | 条件不满足或仍在中间位置 | 不给方案，建议观望 |

> 只有当需要细分“激进/保守入场”或处理 setup 边界情况时，才读取对应 Playbook 文件。

### 8.3 入场前检查（必做）

在输出交易方案前，默认先按以下简版检查；只有用户要求完整 checklist 明细时，才读取 `references/checklists/pre-trade-checklist.md`。

| 项目 | 规则 |
|------|------|
| ① HTF 方向 | 与交易方向一致才优先；冲突则不做，震荡则降仓 |
| ② 位置 | 在支撑附近做多、阻力附近做空；中间位置不做 |
| ③ Setup | 必须匹配 1 个 setup，且核心条件满足 |
| ④ 触发 | 必须有明确触发，不接受“感觉差不多” |
| ⑤ R:R | 到 T1 至少 `1.5:1` |
| ⑥ 事件 | 日内未来 4 小时内有重大事件则不做 |
| ⑦ 否决理由 | 至少写出 1 条潜在失败理由并评估权重 |

### 8.4 仓位与风控（内置简版）

默认先用下列规则，不必先打开 `references/risk/position-sizing.md`：

- 仓位公式：`仓位 = 风险金额 ÷ 止损距离`
- 正常单笔风险：总资金的 `1%–2%`
- 降仓交易：总资金的 `0.5%–1%`
- 绝对上限：总资金的 `3%`
- 若有多个 `⚠️`，每个 `⚠️` 让仓位再乘以 `50%`
- 若用户未提供总资金，只给风险百分比和公式，不虚构具体手数

> 只有当用户明确提供资金规模、杠杆、合约面值并要求精确仓位时，才读取 `references/risk/position-sizing.md`。

### 8.5 方案格式

```
📊 交易方案
├── Setup: [匹配的 Playbook 名称]
├── 方向: 做多 / 做空
├── 入场条件: [具体描述何时入场]
├── 入场价: [价格或条件]
├── 止损位: [价格] （理由：xxx）
├── 目标位1: [价格] （风险回报比 x:1）
├── 目标位2: [价格] （可选，更远目标）
├── 仓位建议: [按风险百分比与止损距离公式计算]
├── 失效条件: [什么情况下方案失效]
└── 持仓管理: [默认 T1 减仓 50% + 止损移至成本价；若趋势延续则跟踪止损]
```

## 标准输出模板

AI 完成分析后，应按以下格式输出：

```

#### 基础信息
```

## AI 注意事项

1. **不要跳步**：必须从 Step 0 开始，逐步分析
2. **先分析后结论**：先完成 Step 0~7，再输出 Step 8；不要边分析边下方向或点位
3. **承认看不清**：如果图片模糊或信息不完整，主动向用户确认
4. **不要过度解读**：看不到的信号不要猜，宁可说"无法判断"
5. **数据优先**：同一问题同时有图片和数据时，以结构化数据为准
6. **图片保守**：只有截图且 K 线细节不清时，不做微观 K 线强判断
7. **最小读取**：不要为了走流程机械打开所有 references，默认用本 workflow 内置规则
8. **概率表述**：用"倾向于"、"可能"、"大概率"，不用"一定会"
9. **风控必须有**：即使用户没问，交易方案也必须包含止损
10. **多时间框架提醒**：如果只看到一个时间框架，提醒用户确认更大时间框架的方向
11. **禁止编造价位**：如果价格刻度不可读或数据不足，绝对不能编造具体价格数字，只能用条件描述
12. **免责声明（强制）**：每次输出分析报告末尾必须附加免责声明
13. **多图先看高周期**：同品种多周期分析时，必须先以 HTF 定方向，再用 LTF 找触发；若冲突则观望
14. **必须匹配 Playbook**：Step 8 输出方案前，必须先匹配 setup；不匹配则不给方案
15. **必须过 Checklist**：匹配 Playbook 后，必须先完成检查，全部通过才给方案
16. **仓位必算**：交易方案中必须给出风险百分比或按公式反推的仓位建议
17. **持仓管理提醒**：方案末尾默认给出 T1 减仓和移止损规则；只有用户追问细节时再读取 `references/checklists/in-trade-management.md`
