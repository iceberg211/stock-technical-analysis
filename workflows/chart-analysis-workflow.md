# 图表分析工作流 (Chart Analysis Workflow)

> **当用户发送一张或多张（同品种多周期）K 线图，或要求使用 MCP / OHLC 数据分析同一品种时，AI 必须按以下流程逐步分析，不可跳步。**

## 执行原则

1. **先分析，后结论**：必须先完成 Step 0~7，再输出任何方向、点位、交易方案。
2. **主动获取数据**：如果用户要求分析某标的但未提供任何图表或数据，AI **必须优先**利用可用的 MCP Server 搜索或拉取该标的的最新 OHLC 价格数据，待获取后再执行 Step 0。若无法获取，须告知用户补充图表。
3. **数据优先**：信息可信度优先级为 `MCP / OHLC 数据 > 用户明确给出的数值 > 清晰截图 > 模糊截图`。
4. **最小读取原则**：默认使用本 workflow 内置规则，不要为每一步机械打开 `references/`；只有遇到边界情况时才读取单个相关文件。
5. **图像保守原则**：只有截图时，若价格刻度或最近 3~5 根 K 线细节不清晰，只做结构和区域判断，不做微观 K 线强判断。

---

## 分析流程

```
收到输入（K线图 / MCP-OHLC数据 / 二者皆有）
  │
  ├── Step 0: 分析模式识别
  ├── Step 1: 基础信息识别
  ├── Step 1.5: 多时间框架映射（仅多图时）
  ├── Step 2: 市场结构判断
  ├── Step 3: 关键支撑阻力标记
  ├── Step 4: 当前价格行为分析
  ├── Step 5: 形态识别
  ├── Step 6: 指标信号（如图上可见）
  ├── Step 7: 综合研判
  └── Step 8: 交易方案（如有信号）
```

---

## Step 0: 分析模式识别

先识别本次分析的数据来源，并据此决定能否给出具体价位与微观判断。

| 模式 | 数据来源 | 允许输出 |
|------|---------|---------|
| **数据模式** | MCP / OHLC / API 返回的结构化行情 | 可做结构分析、关键位分析、最近 K 线分析、具体条件价位 |
| **图片模式** | 用户仅提供截图 | 以结构、区域、明显形态为主；价格刻度不清时禁止输出具体价位 |
| **混合模式** | 同时有截图和结构化数据 | 以数据为准，截图用于辅助交叉验证 |

输出：`分析模式: [数据模式 / 图片模式 / 混合模式]`

### MCP / OHLC 数据获取规范

当需要主动拉取结构化数据时，按以下规范操作：

**期望数据字段**（按优先级排列）：

| 字段 | 必需 | 说明 |
|------|------|------|
| timestamp | 是 | K 线时间戳（UTC） |
| open, high, low, close | 是 | OHLC 四价 |
| volume | 推荐 | 成交量（无则跳过量能分析） |
| symbol | 是 | 标的代码 |
| interval | 是 | K 线周期（1m/5m/15m/1h/4h/1d/1w） |

**可选扩展字段**（如 MCP 提供）：

| 字段 | 说明 |
|------|------|
| open_interest | 持仓量（期货/加密衍生品） |
| funding_rate | 资金费率（加密永续合约） |
| vix | VIX 波动率指数 |

**数据量要求**：至少 50 根 K 线（用于结构判断）；200 根以上为佳（用于均线和形态识别）。

**失败回退协议**：

| 失败场景 | 回退行动 |
|---------|---------|
| MCP 不可用或调用失败 | 告知用户"数据服务暂时不可用"，请求用户上传截图 |
| 返回数据不足 50 根 | 用可用数据做有限分析，在结论中标注"数据量不足，信心降级" |
| 缺少 volume 字段 | 跳过所有量能相关分析，注明"量能数据不可用" |
| 数据明显异常（如价格为 0 或时间戳混乱） | 丢弃异常数据，告知用户数据质量问题 |
| 用户要求的标的不在 MCP 覆盖范围 | 告知用户该标的暂不支持，请求上传截图 |

## Step 1: 基础信息识别

**先看清楚图的基本信息，不要急于判断方向。**

| 识别项 | 说明 |
|--------|------|
| 品种 | 什么资产？（股票/期货/加密货币） |
| 图表数量 | 一张还是多张？是否为同一交易品种？ |
| 时间框架 | 什么级别？（1分钟/15分钟/1小时/4小时/日线/周线） |
| 可见K线数量 | 大约能看到多少根K线？ |
| 数据来源 | 截图 / MCP / OHLC / 混合 |
| 图上指标 | 有没有叠加 RSI/MACD/均线/布林带等？ |
| 当前价格 | 图表最右侧的当前价格大约在什么位置？ |

> 如果图上信息不清楚，**主动向用户确认**，不要猜测。

### 市场类型适配

根据 Step 1 识别的品种，在后续分析中自动适配：

| 市场类型 | 适配要点 |
|---------|---------|
| 股票 | 注意开盘跳空、盘前盘后、财报日历、交易时间限制 |
| 期货 | 注意合约到期、交割月切换、保证金变动 |
| 加密货币 | 24/7 交易无跳空、波动率更高、资金费率影响、无涨跌停限制 |
| 外汇 | 交易时段（亚/欧/美）影响波动性、注意利率决议 |

> AI 不需要在输出中专门写一段"市场适配说明"，而是在后续 Step 2-8 中自然融合上述要点。

### 硬性门槛（不满足则停止）

| 条件 | 行动 |
|------|------|
| 无法识别品种或时间框架 | ❌ 停止分析，向用户确认 |
| 多张图但品种不一致 | ❌ 停止合并分析，要求用户按品种拆分 |
| 价格刻度不可读/模糊 | ⚠️ 可继续到 Step 7 给出方向判断，但 **Step 8 禁止输出具体价格**，只给条件描述 |
| 只有截图且最近 3~5 根 K 线细节不清晰 | ⚠️ Step 4 只能输出“粗略动量判断”，不能输出长影线/吞没/衰竭等强结论 |
| 图片严重模糊/截断 | ❌ 停止分析，请用户重新截图 |

## Step 1.5: 多时间框架映射（仅多图/多周期数据时）

**仅在同一品种有 2 个及以上不同周期的图或数据时执行。**

### 1.5.1 角色分配（先大后小）

根据可用周期数量，分配角色：

**两层模式**（2 个周期）：

- **HTF（高周期）**：定方向与关键区间
- **LTF（低周期）**：找触发与风控细节

**三层模式**（3 个周期，推荐）：

- **Direction（方向层）**：最高周期，只负责定趋势方向，不看 K 线细节
- **Signal（信号层）**：中间周期，负责结构判断、关键位标记、Playbook 匹配
- **Entry（入场层）**：最低周期，负责精确入场时机和止损定位

> 三层模式下，Step 2~5 主要在 Signal 层执行，Direction 层只提供方向过滤，Entry 层只在 Step 8 入场时使用。

### 1.5.2 常见周期组合（示例）

| Direction（定方向） | Signal（找信号） | Entry（精入场） |
|-------------------|-----------------|----------------|
| 周线 (1W) | 日线 (1D) | 4 小时 (4H) |
| 日线 (1D) | 4 小时 (4H) | 1 小时 (1H) |
| 4 小时 (4H) | 1 小时 (1H) | 15 分钟 (15m) |
| 1 小时 (1H) | 15 分钟 (15m) | 5 分钟 (5m) |

> 只有两个周期时，按两层模式处理（HTF = Direction + Signal，LTF = Entry）。

### 1.5.3 一致性检查（硬性）

| 场景 | 处理规则 |
|------|---------|
| Direction 偏多，Signal/LTF 出现看多触发 | ✅ 允许给做多方案 |
| Direction 偏空，Signal/LTF 出现看空触发 | ✅ 允许给做空方案 |
| Direction 与 Signal/LTF 方向冲突 | ❌ 不给具体方案，先观望等待低周期回归高周期方向 |
| Direction 震荡/混乱 | ⚠️ 降低信心等级，优先等待突破后再评估 |
| Signal 与 Entry 冲突（三层模式） | ⚠️ 降仓处理，以 Signal 层结论为主 |

**输出**：`多周期映射: [Direction=xxx, Signal=xxx, Entry=xxx] / 一致性: [一致 / 冲突 / 中性]`
（两层模式时输出：`多周期映射: [HTF=xxx, LTF=xxx] / 一致性: [一致 / 冲突 / 中性]`）

## Step 2: 市场结构判断

**这是最重要的一步。先定方向，再看信号。**

默认按本节内置规则执行；只有在用户明确追问 BOS / CHoCH 定义或结构争议较大时，才读取 `references/core/market-structure.md`。

### 2.1 标记主要 Swing 高低点

- 找到图表中最明显的 **Swing High** 和 **Swing Low**
- 只标记 Major Swing（一眼能看到的），忽略 Minor Swing
- **Major vs Minor 判断标准**：Major Swing 的推动段至少包含 3 根以上同向 K 线，且价格变动幅度在当前分析范围内（或图表可见范围内）高低点绝对落差中占比 >= 10%；不满足的视为 Minor Swing，忽略

### 2.2 判断市场状态

| 状态 | 特征 | 结论 |
|------|------|------|
| **上升趋势** | Higher Highs + Higher Lows | 只找做多机会 |
| **下降趋势** | Lower Highs + Lower Lows | 只找做空机会 |
| **震荡区间** | 最近 3 个以上 Swing High 处于同一价格带（极差 <= ATR x 1.5 或区间高度的 5%），Swing Low 同理 | 区间高抛低吸 或 等待突破 |
| **混乱/过渡** | 高低点既不形成 HH/HL 或 LH/LL，也不走平；或连续出现 BOS 后立即 CHoCH | ❌ 不建议交易 |

### 2.3 检查 BOS / CHoCH

- 最近是否有 BOS（趋势延续确认）？
- 最近是否有 CHoCH（趋势可能变化的预警）？
- 如有 CHoCH，是否已有反向 BOS 确认反转？

### 2.4 评估趋势健康度

- **Impulse（推动浪）判断**：同向 K 线占比 >= 70% + 平均实体 >= 近期 K 线平均实体的 1.5 倍
- **Pullback（回调浪）判断**：实体小（< 推动浪平均实体）+ 颜色混杂（多空 K 线交替）+ 时间 >= 推动浪 K 线数量的 50%
- **健康回调**：回调深度在 Fib 0.382~0.618 之间 + 未破坏前一个 Major Swing + 无反向动量 K 线（实体 >= 推动浪 K 线的等量级）
- **动量衰竭信号**：最近一次推动浪的幅度 < 前一次推动浪的 60%，或推动浪 K 线实体逐根缩小

**输出**：`市场状态: [上升趋势 / 下降趋势 / 震荡 / 混乱]`

## Step 3: 关键支撑阻力标记

默认按本节内置规则执行；只有在需要扩展到供需区、翻转位细分时，才读取 `references/core/support-resistance.md`。

### 3.1 找最明显的水平

- 缩小看全局：有哪些价格区域被多次触碰？
- 只标记**最明显的 2~3 个**，不要画满屏幕
- **"明显"的量化标准**：该价格区域至少被 2 次以上的 Swing High/Low 或 K 线密集区触碰；优先选触碰次数最多、时间跨度最大的水平

### 3.2 判断当前价格位置

| 位置 | 定义 | 含义 |
|------|------|------|
| 靠近支撑区域 | 当前价格距支撑 <= 最近支撑到阻力距离的 20% | 关注反弹信号 或 跌破风险 |
| 靠近阻力区域 | 当前价格距阻力 <= 最近支撑到阻力距离的 20% | 关注突破信号 或 回落风险 |
| 在中间位置 | 不满足以上任一条件 | "无人区"，信号模糊，等待靠近关键位 |

### 3.3 检查翻转

- 有没有曾经的支撑被跌破后变成了阻力？
- 有没有曾经的阻力被突破后变成了支撑？

**输出**：`当前位置: [在支撑附近 / 在阻力附近 / 中间无人区]`

## Step 4: 当前价格行为分析

默认按本节内置规则执行；只有在需要细分真假突破、复杂价格行为特征时，才读取 `references/core/price-action.md`。

如果是**图片模式**且最近 3~5 根 K 线不够清晰：

- 只输出：`多头主导 / 空头主导 / 犹豫不决 / 无法细判`
- 不要输出：`长上影拒绝 / 长下影拒绝 / 动量蜡烛 / 衰竭吞没` 这类依赖细节像素的强判断

### 4.1 实体分析

- 最近 3~5 根 K 线的实体是否在逐根变大（动量增强）或逐根变小（衰竭）？
- 多方占主导：最近 5 根 K 线中 >= 3 根为阳线且阳线平均实体 > 阴线平均实体
- 空方占主导：最近 5 根 K 线中 >= 3 根为阴线且阴线平均实体 > 阳线平均实体
- 犹豫不决：多空 K 线交替出现，实体大小接近

### 4.2 影线分析

- **长影线**定义：影线长度 >= 实体的 1.5 倍
- 长影线出现在关键位 = 被拒绝的信号
- 上影线为主（上方有卖压）还是下影线为主（下方有买压）？

### 4.3 动量判断

- **动量蜡烛**定义：单根 K 线实体 >= 前 3 根 K 线平均实体的 2 倍
- 动量方向与趋势方向是否一致？
- 连续 2 根以上动量蜡烛 = 强动量；单根动量蜡烛后缩小 = 可能是衰竭脉冲

**输出**：`近期行为: [多头主导 / 空头主导 / 犹豫不决 / 动量衰竭 / 无法细判]`

## Step 5: 形态识别

默认只识别高频、低歧义形态；不要为了“找形态”而机械打开全部 pattern references。

### 5.1 K线形态检查

- 最近 1~5 根 K线是否构成已知的反转/延续形态？
- 默认只识别：锤子线、射击之星、吞没、启明星、黄昏星
- 如果截图无法清晰区分锤子 / 上吊、射击之星 / 倒锤子，直接写 `无明显形态`
- 只有用户明确追问某个复杂 K 线形态时，才读取 `references/patterns/candlestick-patterns.md`

### 5.2 图表形态检查

- 更大范围内是否在形成图表形态（双底、头肩、旗形、三角形等）？
- 形态是否已完成？还是正在形成中？
- 默认只识别：双顶/双底、头肩/倒头肩、旗形、三角形
- 只有用户明确追问复杂图表形态时，才读取 `references/patterns/chart-patterns.md`

### 5.3 信号强度评估

如果识别到形态，逐项加分：

| 因素 | 分值 | 说明 |
|------|------|------|
| 形态本身成立 | +1 | 基础分 |
| 出现在关键位（Step 3 标记的支撑/阻力附近） | +1 | 位置加分 |
| 与趋势方向一致（Step 2 判断的趋势同向） | +1 | 方向加分 |
| 指标确认（RSI 背离/超买超卖、MACD 同向） | +1 | 指标加分 |
| 成交量确认（形态处放量，或突破放量） | +1 | 量能加分 |

| 评级 | 总分 | 行动 |
|------|------|------|
| **强信号** | >= 4 分 | 可输出交易方案 |
| **中等信号** | 2~3 分 | 可输出交易方案，但标注信心为"中等" |
| **弱信号** | <= 1 分 | 不输出交易方案，仅建议观望 |

**输出**：`识别形态: [形态名称] / 信号强度: [弱/中/强] / 无明显形态`

---

## Step 6: 指标信号（如图上可见）

默认只做轻量指标判断，不要为了补全流程而读取全部指标资料。

如果图表上叠加了指标，或结构化数据中已提供可直接判断的指标，再分析：

### RSI（如可见）
- 当前 RSI 位置（超买/超卖/中性）
- 是否存在背离？
- RSI 与趋势方向是否一致？

### MACD（如可见）
- MACD 线位置（零轴上方/下方）
- 是否存在背离？
- 柱状图是否在扩大/缩小？

### 其他指标
- 均线排列方向？
- 布林带收缩/扩张？

> 如果图上没有指标、数据中也未提供指标，跳过此步骤，不要凭空猜测。
> 只有用户明确要求 RSI / MACD 的详细解释时，才读取 `references/indicators/` 下对应文件。

**输出**：`指标确认: [确认趋势 / 出现背离 / 无指标可判断]`

---

## Step 7: 综合研判

将 Step 2~6 的结果汇总，给出综合判断：

> 本步骤先产出“分析结论”，不要抢先输出交易决策卡或具体价位。

### 多空偏向

| 条件组合 | 偏向 |
|---------|------|
| 上升结构 + 支撑附近 + 看涨形态 | 🟢 偏多 |
| 下降结构 + 阻力附近 + 看跌形态 | 🔴 偏空 |
| 震荡 + 关键位 + 反转形态 | 🟡 区间交易 |
| 结构不清 / 无明确形态 / 中间位置 | ⚪ 观望 |

### 信心等级

| 等级 | 条件 |
|------|------|
| **高信心** | 结构清晰 + 关键位 + 形态确认 + 指标确认（4重汇合） |
| **中等信心** | 3 个因素汇合 |
| **低信心** | 仅 1~2 个因素 → 建议观望或等待更多确认 |

### 多时间框架一致性（多图时必做）

#### 三层 MTF 框架（推荐）

当用户提供 3 个时间框架或数据覆盖 3 个周期时，使用三层分析：

| 层级 | 作用 | 典型周期示例 |
| ------ | ------ | ------------ |
| **Direction（方向层）** | 定大趋势，只看不做 | 周线 / 日线 |
| **Signal（信号层）** | 找 setup 和关键位 | 4H / 1H |
| **Entry（入场层）** | 精确入场时机 | 15m / 5m |

**周期搭配建议**（可按用户习惯调整）：

| 风格 | Direction | Signal | Entry |
| ------ | --------- | ------ | ----- |
| 波段交易 | 周线 | 日线 | 4H |
| 短线交易 | 日线 | 4H | 1H |
| 日内交易 | 4H | 1H | 15m |
| 超短线 | 1H | 15m | 5m |

**三层一致性判定**：

| Direction | Signal | Entry | 最终偏向 |
| --------- | ------ | ----- | --------- |
| 偏多 | 看多 setup | 看多触发 | 🟢 偏多（可执行） |
| 偏空 | 看空 setup | 看空触发 | 🔴 偏空（可执行） |
| 偏多 | 看多 setup | 无触发 | ⏳ 等 Entry 层触发 |
| 偏多 | 看空 setup | 任意 | ⚪ 冲突，观望 |
| 偏空 | 看多 setup | 任意 | ⚪ 冲突，观望 |
| 震荡/混乱 | 任意 | 任意 | 🟡 降级处理，谨慎 |

> 核心原则：Direction 定方向 -> Signal 找机会 -> Entry 选时机。三层同向才执行。

#### 两层 MTF（简化版）

当用户只提供 2 个时间框架时，退化为两层：

| HTF 结论 | LTF 触发 | 最终偏向 |
|---------|---------|---------|
| 偏多 | 看多触发 | 🟢 偏多（可执行） |
| 偏空 | 看空触发 | 🔴 偏空（可执行） |
| 偏多 | 看空触发 | ⚪ 冲突，观望 |
| 偏空 | 看多触发 | ⚪ 冲突，观望 |
| 震荡/混乱 | 任意触发 | 🟡 降级处理，谨慎 |

#### 单时间框架

当用户只提供 1 个时间框架时，在该框架内完成 Step 2~6，默认在信心等级上降一档（因缺少多周期确认）。

**例外**：如果该单一框架内结构极其清晰（趋势 + 关键位 + 形态 + 指标四重汇合全部满足），可以在明确提示"仅基于单一时间框架，建议确认更大周期方向"的前提下，保持原信心评级，不强制降档。

## Step 8: 交易方案

完成 Step 7 综合研判后，根据信号强度决定下一步：

| 信号强度 | 行动 |
| -------- | ------ |
| **强/中** | 👉 **强制流转**：AI 必须立即调用工具读取 `workflows/trading-decision.md` 继续后续分析（Playbook 匹配 -> Checklist -> 仓位），并将决策结果无缝衔接在当前输出后。 |
| **弱** | 不输出交易方案，只给观察建议 |
| **观望** | 明确说”当前不满足交易条件，建议观望” |

> **强制自查 (Sanity Check)**：在进入交易决策流程前，务必默默自查：1) 信号方向是否与大周期趋势冲突？2) 是否存在事件冲突等硬性否决项？如有冲突则直接观望，不进入决策流程。

👉 **输出格式限制**：必须读取 `workflows/output-templates.md`，严格遵守其模板结构，并根据对话场景选择完整模式、精简模式或快快答模式构建输出。
