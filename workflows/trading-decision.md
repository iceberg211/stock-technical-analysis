# 交易决策流程 (Trading Decision Process)

> **本文件在 Step 7 综合研判完成后、需要输出交易方案时读取。**
> 弱信号或观望结论不需要读取本文件。

---

## 8.1 前置检查（硬性门槛）

| 条件 | 行动 |
| ------ | ------ |
| 价格刻度不可读 | **禁止输出具体入场价/止损价/目标价**，改为描述条件（如"突破前高时入场""形态最低点下方止损"） |
| 信号强度为"弱" | 不输出交易方案，只给观察建议 |
| 市场状态为"混乱" | 不输出交易方案，建议观望 |
| 多周期一致性为"冲突" | 不输出交易方案，等待低周期重新与高周期同向 |

## 8.2 Playbook 匹配与执行规则

默认先按下表快速匹配；所有入场/止损/目标信息已内联，无需额外读取文件。

| Setup | 最少条件 | 止损 | 目标 | 特殊否决 |
| ------ | --------- | ------ | ------ | ------ |
| `trend-pullback` | 趋势清晰（≥2 次同向 BOS）+ 无 CHoCH + 回调到关键位 + 出现反转信号 | 回调极值外 + buffer | T1=前高/前低，T2=下一关键位 | 趋势已走 3 波以上；HTF 冲突；回调超 0.786 |
| `breakout-retest` | 动量实体突破关键位（实体≥前3根2倍）+ 收盘越过 + 回踩附近 + 拒绝 PA | 回踩极值外 + buffer | T1=等距投射（`突破位 + (突破位-前摆动低点)`），T2=下一关键位 | 突破方向与 HTF 冲突；区间被来回穿多次 |
| `range-reversal` | 成熟区间（≥4次触碰+≥20根K线）+ 价格靠近边缘 20% + 拒绝/反转信号 | 区间边沿外 + buffer | T1=区间中轴，T2=区间对侧 | 区间内有 BOS；中间位置；区间越来越窄（换用 chart-patterns） |
| `false-breakout-reversal` | 关键位被穿 + 1~3 根 K 线内**实体**收回内侧 + 长影线（≥实体1.5倍）+ 出现反向触发（Pin Bar/吞没/低级别反转） | **假突破极值外** + buffer（不是回调低点） | T1=结构/区间对侧关键位，T2=更远结构位或等距 | 突破后已走3根以上同向动量K线（是真突破回调）；突破方向与 HTF 一致 |
| `flag-wedge-breakout` | 明确前置 Impulse（旗杆≥3根动量K线）+ 旗面/楔形（≥5根K线）+ 向趋势方向动量突破 + 收盘越过边界 | 形态内最远极值外 + buffer | T1=旗杆等距投射（`突破点 + 旗杆长度`）；楔形以形态最宽处投射 | 趋势走3波以上+楔形出现=优先视为反转；形态持续>30根K线 |
| `无匹配` | 上述条件均不满足，或价格在中间位置 | — | — | 不给方案，建议观望 |

> buffer 通用规则：首选 `ATR × 0.3~0.5`；无 ATR 时取极值点影线长度的 20%，或止损位的 `0.5~1%` 作固定偏移。

## 8.3 入场前检查（必做）

在输出交易方案前，默认先按以下简版检查；只有用户要求完整 checklist 明细时，才读取 `references/checklists/pre-trade-checklist.md`。

| 项目（含原ID） | 规则 |
| ------ | ------ |
| HTF 方向（ID:1） | 与交易方向一致才优先；冲突则不做，震荡则降仓 |
| 位置（ID:2） | 在支撑附近做多、阻力附近做空；中间位置不做 |
| Setup 匹配（ID:3） | 必须匹配 1 个 setup，且核心条件满足 |
| 触发信号（ID:4） | 必须有明确触发，不接受"感觉差不多" |
| 盈亏比 R:R（ID:5） | 到 T1 至少 `1.5:1` |
| 事件与宏观（ID:6） | 日内未来 4h 内有重大事件则不做；如有宏观数据（大盘方向/VIX/资金费率），与交易方向冲突则降仓或不做 |
| 否决理由检查（ID:7） | 至少写出 1 条潜在失败理由并评估权重 |

**输出规范（强制）**：

- 禁止只输出编号（如 `3 ✅ / 4 ⚠️ / 5 ✅`）
- 每项必须写为：`项目名: [✅ / ⚠️ / ❌]（一句原因）`
- 硬否决项（Setup/触发/R:R）出现任意 `❌` -> 直接 `不做`
- 软降级项出现 `⚠️` -> 允许做，但必须明确降仓比例

## 8.4 仓位与风控（内置简版）

默认先用下列规则，不必先打开 `references/risk/position-sizing.md`：

- 仓位公式：`仓位 = 风险金额 / 止损距离`
- 正常单笔风险：总资金的 `1%~2%`
- 降仓交易：总资金的 `0.5%~1%`
- 绝对上限：总资金的 `3%`
- 若有多个降级项，每个降级项让仓位再乘以 `50%`
- 若用户未提供总资金，只给风险百分比和公式，不虚构具体手数

> 只有当用户明确提供资金规模、杠杆、合约面值并要求精确仓位时，才读取 `references/risk/position-sizing.md`。

## 8.7 复盘与统计闭环

### 1. 单笔复盘（当用户提供老方案并要求复盘时）

无需拘泥于具体表格形式，按以下 **5 个维度** 输出复盘文字：

1. **原貌摘要**：原定的入场/止损/目标价。
2. **实际触发**：是否触发？最终达到哪一级目标或止损？
3. **对比分析**：客观对比"原结构判断 vs 实际结构走势"、"原信号 vs 实际信号"。
4. **纪律评分**：指出是否按 Playbook、Checklist 和规则仓位操作。违规即使赚钱也是坏交易。
5. **改进建议**：指出具体的执行偏差与改进点。

### 2. 批量统计（当用户给定多笔记录时）

汇总胜率，并输出紧凑清单：

1. **总览**：总交易 N 笔，胜 x 笔，负 x 笔，未触发 x 笔，综合胜率 xx%。
2. **按 Playbook 拆解**：列出每个 setup 的胜率与平均盈亏比 (R)。
3. **常见死因 Top 3**：总结导致亏损最多的 3 种具体场景。
4. **规则补丁**：针对低胜率环境，提供后续过滤建议。
