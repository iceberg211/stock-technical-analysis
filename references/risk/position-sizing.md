# 仓位计算与风控规则 (Position Sizing)

> **仓位大小不是"感觉"出来的，是由"你愿意亏多少"加"止损在哪"反推出来的。**
> 这是把"分析正确"变成"账户安全"的核心环节。

---

## 一、核心公式

```
仓位 = 风险金额 ÷ 止损距离

其中：
  风险金额 = 总资金 × 单笔风险百分比
  止损距离 = |入场价 - 止损价|
```

> 这个公式的含义：**不管什么品种、什么价格，每笔交易的最大亏损金额是固定的。**

---

## 二、单笔风险上限

| 参数 | 标准值 | 说明 |
|------|-------|------|
| **正常单笔风险** | 总资金的 **1–2%** | 大多数交易适用 |
| **高信心交易** | 最多 **2%** | 4 重汇合 + setup 完美 |
| **降仓交易** | **0.5–1%** | HTF 不确定 / 逆势 / 事件日 / 连亏后 |
| **绝对上限** | **3%** | 任何情况不得超过 |

### 示例计算

```
总资金: $10,000
单笔风险: 1% = $100
入场价: $50.00
止损价: $48.00
止损距离: $2.00

仓位 = $100 ÷ $2.00 = 50 股

验证: 如果被止损 → 亏损 = 50 × $2.00 = $100 = 总资金的 1% ✅
```

### 加密货币 / 合约示例

```
总资金: $5,000（交易账户，非全部身家）
单笔风险: 1% = $50
入场价: $60,000 (BTC)
止损价: $58,500
止损距离: $1,500

仓位 = $50 ÷ $1,500 = 0.033 BTC（约 $2,000 名义价值）

注意: 如果使用杠杆，仓位不变，只是保证金减少。
      不要因为有杠杆就加大仓位！
```

## 三、降仓规则

### 什么时候必须降仓？

| 情况 | 仓位调整 | 原因 |
|------|---------|------|
| HTF 方向不明/震荡 | 正常仓位 × 50% | 方向不确定性高 |
| 逆势交易（允许的情况下） | 正常仓位 × 50% | 逆势胜率低 |
| 事件日交易 | 正常仓位 × 50% | 波动可能放大 |
| 连亏后（见第四节） | 按连亏规则自动降 | 保护资金 |
| R:R 在 1.5:1–2:1 之间 | 正常仓位 × 75% | 盈亏比一般 |
| "不回踩例外"激进入场 | 正常仓位 × 50% | 非标准 setup |
| 加密货币使用高杠杆（>5×） | 名义仓位不变，但格外注意保证金比例 | 杠杆放大风险 |

### 不降仓的情况

| 情况 | 仓位 |
|------|------|
| HTF 同向 + setup 完美 + R:R ≥ 2:1 | 正常仓位（1–2%） |
| 高信心 4 重汇合 | 可用仓位上限（2%），不超过 |

## 四、连续亏损处理

> **连续亏损不只是运气差，也可能是环境变了。降速是为了活下来。**

### 连亏阶梯

| 连亏笔数 | 行动 | 原因 |
|---------|------|------|
| 连亏 **2 笔** | ⚠️ 自我检查：是 setup 问题还是执行问题？ | 早期自省 |
| 连亏 **3 笔** | 🔴 仓位自动降至 **正常的 50%** | 保护资金 |
| 连亏 **5 笔** | 🔴🔴 **暂停交易一天**（或一个交易时段） | 冷静 + 复盘 |
| 连亏 **7 笔** | 🔴🔴🔴 暂停交易 + 完整复盘所有亏损单 + 请求外部反馈 | 可能有系统性问题 |

### 恢复规则

| 条件 | 行动 |
|------|------|
| 暂停后恢复 | 前 3 笔用 **50% 仓位**，3 笔盈利后恢复正常 |
| 连亏后首笔盈利 | 不立即恢复满仓，需要连续 2 笔盈利才恢复 |

> 🔑 **连亏后最危险的不是继续亏，是情绪上头后加大仓位想"一把回来"。**

---

## 五、日最大亏损（当日熔断）

| 参数 | 标准值 | 行动 |
|------|-------|------|
| **日最大亏损** | 总资金的 **3–5%** | 达到后**当日停止交易** |
| **周最大亏损** | 总资金的 **8–10%** | 达到后**本周停止交易** |

> 这是"熔断机制"。不管还有多好的机会，到了上限就必须停。

---

## 六、相关性风控

> **同方向 + 同品种/高度相关品种 = 重复暴露 = 风险翻倍。**

| 规则 | 说明 |
|------|------|
| 同一品种 | 同一品种最多 **1 笔** 持仓 |
| 高相关品种 | 相关品种（如 BTC/ETH、黄金/白银、纳指/标普）同方向最多 **2 笔** |
| 总暴露上限 | 所有持仓的总风险不超过总资金的 **5%** |

### 常见高相关组

| 组别 | 品种 |
|------|------|
| 加密主流 | BTC, ETH |
| 美股大盘 | SPY, QQQ, IWM |
| 贵金属 | 黄金, 白银 |
| 原油 | WTI, Brent |
| 中概股 | 同行业内的港股/A股 |

---

## 七、杠杆使用规则

| 规则 | 说明 |
|------|------|
| 杠杆不改变仓位逻辑 | 杠杆只影响保证金需求，**不应该**用来放大仓位 |
| 名义仓位不变 | 无论用多少杠杆，按本文公式算出的仓位（股数/合约数/BTC数）不变 |
| 保证金要求 | 保证金比例保持 > 2 倍维持保证金，留足安全垫 |
| 高杠杆警告 | 杠杆 > 10× → 极度危险，必须确保止损是硬止损（不能扛） |

> ❌ **"有 10 倍杠杆所以仓位开 10 倍"= 自杀式交易。**

## AI 使用指引

- 给出交易方案时，**必须**附上仓位建议（按公式计算）
- 如果用户提供了总资金，直接算出具体仓位大小
- 如果用户没提供总资金，给出公式 + 百分比建议
- 如果用户提到连亏 → 先检查连亏处理规则，再讨论新交易
- 如果用户想满仓 / 重仓 → 明确拒绝，引用单笔风险上限
- 如果用户使用杠杆 → 提醒杠杆不改变仓位逻辑
- 仓位计算结果应以"如果被止损，亏损 = 总资金的 X%"格式验证
