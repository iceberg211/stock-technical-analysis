# 持仓管理规则 (In-Trade Management)

> **入场只是交易的开始。大多数人亏钱不是因为方向错了，而是持仓管理失控。**
> 这份文件定义：入场后做什么、什么时候动止损、什么时候走。

---

## 一、入场后立即执行

| 操作 | 说明 | 硬性/建议 |
|------|------|----------|
| 确认止损已设 | 检查止损单是否已下达/记录 | **硬性** |
| 确认仓位正确 | 仓位是否按 `risk/position-sizing.md` 计算 | **硬性** |
| 记录入场信息 | 品种、方向、入场价、止损价、目标价、对应 Playbook | 建议 |
| 设定提醒 | 在 T1 / 关键位设置价格提醒 | 建议 |

> ❌ **止损没设 = 这笔交易是赌博，不是交易。**

---

## 二、持仓阶段管理（按价格进展）

### 阶段 1: 入场 → 尚未到 T1

| 规则 | 说明 |
|------|------|
| **不动止损** | 保持原始止损位不变，除非出现结构性变化 |
| **不加仓** | 入场后不追加仓位 |
| **不提前平** | 除非出现"提前离场"信号（见第四节） |
| **不看太频繁** | 低于入场时间框架的小波动忽略 |

> 🔑 **这个阶段的核心是：什么都不做。忍住。**

### 阶段 2: 价格到达 T1

默认规则（适用于大多数 Playbook）：

| 操作 | 说明 |
|------|------|
| **减仓** | 平掉 **50%** 的仓位，锁定部分利润 |
| **移动止损** | 剩余仓位的止损移至 **成本价**（保本止损） |
| **更新记录** | 记录 T1 到达时间和价格 |

> T1 到达后，这笔交易已经"不会亏钱"。剩余仓位是"免费彩票"。

#### Playbook 差异化管理

不同 Playbook 在 T1 的管理策略有所不同：

| Playbook | T1 减仓比例 | 止损移动 | 理由 |
|----------|-----------|---------|------|
| **trend-pullback** | 30% | 移至成本价 | 顺势单利润空间大，保留更多仓位让利润奔跑 |
| **breakout-retest** | 50% | 移至成本价 | 标准管理，回踩后突破延续性中等 |
| **flag-wedge-breakout** | 30% | 移至成本价 | 中继形态突破通常有旗杆等距投射的空间 |
| **range-reversal** | 50% | 移至成本价 | 区间利润有限（T2 = 对侧沿），早锁利 |
| **false-breakout-reversal** | 50% | 移至成本价 | 反向单不确定性较高，早锁利 |

### 阶段 3: T1 → T2 之间

| 规则 | 说明 |
|------|------|
| **跟踪止损** | 每当价格创造同方向的新 Swing，止损跟移到前一个 Swing 之后 |
| **让利润奔跑** | 趋势结构未被破坏，不主动平仓 |
| **观察动量** | 如果动量明显衰竭（K 线越来越小 + 影线越来越多），但结构未破，仍然持有 |

#### T1 → T2 的 Playbook 差异

| Playbook | 特殊规则 |
|----------|---------|
| **trend-pullback** | 趋势未破坏（无 CHoCH）时持续跟踪，可考虑 T3 |
| **flag-wedge-breakout** | 到达旗杆等距投射后审视，趋势强则继续跟踪 |
| **range-reversal** | 接近对侧沿时主动减仓或全平，不等 T2 精确触碰 |

### 阶段 4: 到达 T2 或更远

| 操作 | 说明 |
|------|------|
| **平掉剩余仓位** | 在 T2 附近考虑全部平仓 |
| **例外** | 仅 trend-pullback 和 flag-wedge-breakout：如果趋势极强（连续 BOS + 无回调迹象），可留 25% 继续跟踪至 T3 |

## 三、移动止损规则（详细）

### 什么时候移？

| 事件 | 止损移至 |
|------|---------|
| T1 到达 | 成本价（入场价） |
| 价格创新的 Swing Low（做多）/ Swing High（做空） | 新 Swing 之后 + buffer |
| 出现同向 BOS | 止损跟移到造成 BOS 的 Swing 之后 |

### 什么时候不移？

| 情况 | 原因 |
|------|------|
| "感觉涨得差不多了" | 不是规则，不移 |
| 小级别的回调 | 正常波动，不移 |
| 价格还没到 T1 | 太早移动会被正常回调扫掉 |

### 绝对禁止

| 操作 | 原因 |
|------|------|
| ❌ 把止损往亏损方向移 | 扩大止损 = 否定入场逻辑 |
| ❌ 删除止损 | 无止损 = 无限风险 |
| ❌ 亏损时加仓摊薄成本 | 这是对抗市场，不是交易 |

## 四、提前离场规则

以下情况出现任一 → **考虑提前平仓或部分减仓**：

| 信号 | 级别 | 行动 |
|------|------|------|
| 反向动量 K 线 | 🔴 强信号 | 立即减仓 50% 或全平 |
| 反向 BOS（CHoCH） | 🔴 强信号 | 全部平仓 |
| 到达强阻力/支撑且出现拒绝 | 🟡 中等信号 | 减仓 50% |
| RSI/MACD 顶背离/底背离 | 🟡 辅助信号 | 收紧止损但不主动平 |
| 突发重大事件（非预期） | 🔴 强信号 | 全部平仓或大幅缩减 |

## 五、时间止损

> **价格不动也是一种信号。** 如果入场后长时间无进展，setup 的时效性在衰减。

### 时间止损标准

| 情况 | 行动 |
|------|------|
| 入场后 **10–15 根 K 线** 无明显进展（未朝 T1 方向推进） | ⚠️ 审视：缩减仓位 50% 或平仓 |
| 入场后 **20+ 根 K 线** 仍在入场价附近横盘 | ❌ 平仓，释放资金和注意力 |
| 价格虽未到 T1，但结构仍健康，且有新信号确认 | ✅ 可继续持有（重置时间计数） |

> ⚠️ 时间止损的 K 线数量参考的是**入场时间框架**的 K 线。
> 例如：4H 图入场 → 10 根 ≈ 40 小时 ≈ 约 2 个交易日。

## 六、特殊场景处理

### 跳空（Gap）

| 情况 | 行动 |
|------|------|
| 有利方向跳空 | 正常管理，如直接跳过 T1 则按 T1 规则处理 |
| 不利方向跳空（跳过止损） | 开盘后立即平仓，不等反弹 |

### 过夜/周末持仓

| 情况 | 行动 |
|------|------|
| 日内交易 | 收盘前必须平仓，不过夜 |
| 波段交易 | 可过夜但仓位不超过正常的 70% |
| 周末（股票/期货） | 已在盈利的仓位可持有；止损附近的仓位减仓 |
| 加密货币 | 24/7 市场无此限制，但重大事件前适用同样原则 |

## 管理规则速查表

```
入场后立即: 确认止损 + 确认仓位 + 记录
    │
    ├── 还没到 T1: 什么都不做，忍住
    │
    ├── 到了 T1（按 Playbook 区分）:
    │     ├── trend-pullback / flag-wedge: 减仓 30% + 止损移至成本价
    │     └── breakout-retest / range / false-breakout: 减仓 50% + 止损移至成本价
    │
    ├── T1 → T2: 跟踪止损 + 让利润奔跑
    │     └── range-reversal: 接近对侧沿主动减仓
    │
    ├── 到了 T2: 考虑全平
    │     └── 例外: trend-pullback / flag-wedge 趋势极强时留 25% 跟踪至 T3
    │
    ├── 提前离场信号？: 反向动量/BOS/强阻力拒绝 → 减仓或全平
    │
    └── 时间止损: 10-15 根无进展 → 审视; 20+ 根 → 平仓
```

## AI 使用指引

- 用户问"现在该怎么办"（已持仓） → 判断当前在哪个阶段，给出对应规则
- 用户问"要不要移止损" → 检查是否满足移动止损的条件
- 用户问"要不要平仓" → 检查是否出现了提前离场信号
- 用户想加仓/扩大止损 → 明确拒绝，引用"绝对禁止"规则
- 用户问"为什么还不涨/跌" → 检查时间止损标准
- 每次给出管理建议时，引用本文件中的具体规则编号
