# 入场前检查单 (Pre-Trade Checklist)

> **这张清单的作用不是帮你找交易，而是帮你过滤掉不该做的交易。**
> 能通过 7 项检查的交易，才值得动钱。

---

## 检查清单

7 项检查分为两类：

| 类别 | 包含项目 | 规则 |
|------|---------|------|
| **硬否决项** | ③ Setup 匹配、④ 触发信号、⑤ 盈亏比 | 只能 ✅ 或 ❌。任何一项 ❌ → **不做** |
| **软降级项** | ① HTF 方向、② 位置、⑥ 事件、⑦ 否决理由 | 允许 ⚠️。⚠️ 不否决交易，但必须**降仓** |

> ❗ **入场条件 = 硬否决项全部 ✅ + 软降级项无 ❌。**
> 软降级项出现 ⚠️ → 仓位按规则降级，但可以做。

---

### ① HTF 方向一致（软降级项）

| 检查 | 标准 |
|------|------|
| 高周期方向 | HTF 趋势方向 **与** 交易方向一致 |
| 单图分析时 | 提醒用户确认更高时间框架方向 |

| 状态 | 判定 |
|------|------|
| HTF 偏多 + 做多 | ✅ |
| HTF 偏空 + 做空 | ✅ |
| HTF 方向与交易方向冲突 | ❌ 不做 |
| HTF 震荡/不明 | ⚠️ 降仓处理（仓位 × 50%） |

---

### ② 位置合理（软降级项）

| 检查 | 标准 |
|------|------|
| 关键位附近 | 价格在已标记的支撑/阻力/供需区附近 |
| 不在"无人区" | 不在关键位中间的模糊地带 |

| 位置 | 判定 |
|------|------|
| 在支撑附近做多 / 在阻力附近做空 | ✅ |
| 在支撑附近做空 / 在阻力附近做多 | ⚠️ 逆势交易，降仓 50% + 需要极强触发 |
| 在两个关键位中间 | ❌ 等靠近关键位再说 |

---

### ③ Setup 匹配（🔴 硬否决项）

| 检查 | 标准 |
|------|------|
| Playbook 匹配 | 当前行情匹配 `playbooks/` 中某个已定义的 setup |
| 条件满足 | 该 Playbook 的所有**核心**前提条件全部满足 |

| 状态 | 判定 |
|------|------|
| 匹配某个 Playbook 且核心条件全满足 | ✅ |
| 匹配但核心条件缺失 | ❌ 条件没凑齐就不做 |
| 不匹配任何 Playbook | ❌ 没有你的 setup 就不做 |

> 注：Playbook 条件分"核心条件"和"增强条件"，详见各 Playbook 文件。核心条件必须满足，增强条件影响信心等级但不否决。

---

### ④ 触发信号（🔴 硬否决项）

| 检查 | 标准 |
|------|------|
| 入场触发 | 出现了对应 Playbook 定义的具体触发 PA |
| 不是"差不多" | 触发必须明确，不是"感觉像" |

| 状态 | 判定 |
|------|------|
| 有明确反转 K 线 / 低级别 BOS / 拒绝信号 | ✅ |
| "感觉到位了"但没有具体 PA | ❌ 等触发出现 |
| 触发出现但与 Playbook 不匹配 | ❌ 触发类型必须对应 |

---

### ⑤ 盈亏比（🔴 硬否决项）

| 检查 | 标准 |
|------|------|
| 风险回报比 | R:R ≥ **1.5:1**（到 T1 的距离 ÷ 止损距离） |
| 计算方式 | R:R = (目标价 - 入场价) ÷ (入场价 - 止损价) |

| R:R | 判定 |
|-----|------|
| ≥ 2:1 | ✅ 良好 |
| 1.5:1 – 2:1 | ✅ 可接受 |
| 1:1 – 1.5:1 | ❌ 不做 |
| < 1:1 | ❌ 绝对不做 |

---

### ⑥ 事件风险（软降级项）

| 检查 | 标准 |
|------|------|
| 重大数据 | 未来 4 小时内（日内交易）或 1 天内（波段交易）没有重大经济数据发布 |
| 财报 | 如果是个股，不在财报发布前后 1 天内 |
| 央行决议 | 不在利率决议 / 货币政策会议前后 |
| 突发事件 | 没有正在发生的地缘政治/黑天鹅事件 |

| 状态 | 判定 |
|------|------|
| 无即将公布的重大事件 | ✅ |
| 有重大事件但距离较远（>24h） | ⚠️ 降仓 50% + 提前设好出场计划 |
| 重大事件即将公布（<4h） | ❌ 等事件公布后再评估 |

> 加密货币 24/7 市场无"收盘"概念，但仍需关注重大链上事件、监管消息等。

---

### ⑦ 否决理由检查（软降级项，反向思维）

> **在入场前，强制找出至少 1 条"不做"的理由。如果完全找不到 → 通过。**

| 检查 | 标准 |
|------|------|
| 列出理由 | 写出 1-3 条"为什么这笔可能亏钱" |
| 评估权重 | 这些理由是否足以否决交易？ |

常见否决理由：

| 否决理由 | 判定 |
|---------|------|
| "趋势可能正在反转" → 有 CHoCH 但无反向 BOS | ⚠️ 降仓 50% |
| "止损距离太大" → R:R 不够 | ❌ 已被 ⑤ 硬否决 |
| "跟前一笔一模一样，那笔刚止损" | ❌ 否决（情绪交易风险） |
| "找不到理由但就是不舒服" | ⚠️ 降仓 50% |
| "完全找不到反对理由" | ✅ 通过 |

---

## 完整输出格式

AI 在 Step 8 给出交易方案前，应先输出以下检查结果：

```
📋 入场前检查

🔴 硬否决项（任一 ❌ → 不做）
├── ③ Setup: ✅ / ❌ [匹配哪个 Playbook]
├── ④ 触发: ✅ / ❌ [具体触发信号]
├── ⑤ R:R: ✅ / ❌ [具体比值]

🟡 软降级项（⚠️ → 降仓，❌ → 不做）
├── ① HTF 方向: ✅ / ❌ / ⚠️ [说明]
├── ② 位置: ✅ / ❌ / ⚠️ [当前位置与关键位的关系]
├── ⑥ 事件: ✅ / ❌ / ⚠️ [近期事件]
├── ⑦ 否决: ✅ / ❌ / ⚠️ [列出的理由及结论]

结论: ✅ 通过 / ❌ 不做 / ⚠️ 通过但降仓
降仓幅度: [如有 ⚠️，各项降仓叠加后的最终仓位比例]
```

> ⚠️ 多项 ⚠️ 降仓叠加规则：每个 ⚠️ 独立将仓位 × 50%。
> 例如：① ⚠️ + ⑥ ⚠️ → 最终仓位 = 正常仓位 × 50% × 50% = 25%。
> 如果叠加后仓位 < 25%，建议直接放弃此笔交易。

---

## AI 使用指引

- Step 8 输出方案前，**必须**先输出上方检查格式
- 硬否决项（③④⑤）任一 ❌ → 方案改为"建议观望"，说明被哪项否决
- 软降级项（①②⑥⑦）出现 ❌ → 也是不做；出现 ⚠️ → 可做但必须降仓
- 多个 ⚠️ 叠加后仓位过低（< 25%） → 建议放弃
- 用户问"这笔能不能做" → 跑一遍 7 项检查，逐项给出结果
- 不要跳过任何一项，即使用户催促
