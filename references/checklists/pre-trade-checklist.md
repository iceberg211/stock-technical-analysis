# 入场前检查单 (Pre-Trade Checklist)

> **这张清单的作用不是帮你找交易，而是帮你过滤掉不该做的交易。**

7 项检查分为两类：

| 类别 | 包含项目 | 规则 |
|------|---------|------|
| **硬否决项** | ③ Setup 匹配、④ 触发信号、⑤ 盈亏比 | 任何一项 ❌ → 不做 |
| **软降级项** | ① HTF 方向、② 位置、⑥ 事件、⑦ 否决理由 | ⚠️ 不否决但必须降仓 |

> 入场条件 = 硬否决项全部 ✅ + 软降级项无 ❌

> 对外输出规范：可保留 `ID` 便于内部映射，但必须优先写项目名与原因；禁止只写编号（如 `3✅/4⚠️/5✅`）。

---

### ① HTF 方向（软降级项）

| 状态 | 判定 |
|------|------|
| HTF 与交易方向一致 | ✅ |
| HTF 方向冲突 | ❌ 不做 |
| HTF 震荡/不明 | ⚠️ 降仓 × 50% |

---

### ② 位置（软降级项）

| 位置 | 判定 |
|------|------|
| 支撑附近做多 / 阻力附近做空 | ✅ |
| 支撑附近做空 / 阻力附近做多 | ⚠️ 逆势，降仓 50% + 需极强触发 |
| 两个关键位中间 | ❌ 等靠近关键位 |

---

### ③ Setup 匹配（🔴 硬否决项）

| 状态 | 判定 |
|------|------|
| 匹配某个 Playbook 且核心条件全满足 | ✅ |
| 匹配但核心条件缺失 | ❌ |
| 不匹配任何 Playbook | ❌ |

---

### ④ 触发信号（🔴 硬否决项）

| 状态 | 判定 |
|------|------|
| 有明确反转K线 / 低级别 BOS / 拒绝信号 | ✅ |
| "感觉到位了"但无具体 PA | ❌ 等触发出现 |
| 触发出现但与 Playbook 不匹配 | ❌ |

---

### ⑤ 盈亏比（🔴 硬否决项）

R:R = (目标价 - 入场价) ÷ (入场价 - 止损价)

| R:R | 判定 |
|-----|------|
| ≥ 2:1 | ✅ 良好 |
| 1.5:1 – 2:1 | ✅ 可接受 |
| < 1.5:1 | ❌ 不做 |

---

### ⑥ 事件风险（软降级项）

| 状态 | 判定 |
|------|------|
| 无即将公布的重大事件 | ✅ |
| 有重大事件但距离 > 24h | ⚠️ 降仓 50% + 提前设好出场计划 |
| 重大事件即将公布（< 4h） | ❌ 等事件公布后再评估 |

---

### ⑦ 否决理由检查（软降级项）

**入场前强制找出至少 1 条"不做"的理由。**

| 否决理由 | 判定 |
|---------|------|
| 有 CHoCH 但无反向 BOS（趋势可能反转） | ⚠️ 降仓 50% |
| 跟前一笔一模一样，那笔刚止损 | ❌ 否决 |
| 找不到理由但就是不舒服 | ⚠️ 降仓 50% |
| 完全找不到反对理由 | ✅ 通过 |

---

## 仓位叠加规则

每个 ⚠️ 独立让仓位 × 50%。例：① ⚠️ + ⑥ ⚠️ → 最终仓位 = 正常 × 25%

> 叠加后仓位 < 25% → 建议直接放弃此笔交易
