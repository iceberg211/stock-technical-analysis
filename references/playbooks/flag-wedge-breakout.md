# 旗形/楔形突破 (Flag & Wedge Breakout)

> **趋势中最经典的中继形态：一段猛烈推动后，价格在旗形或楔形中"喘气"，然后继续原方向。**
> 与 trend-pullback 的区别：trend-pullback 等回调到关键位；本 Playbook 等形态突破确认。

---

## 一、形态识别

### 牛旗 / 熊旗（Flag）

| 特征 | 标准 |
|------|------|
| 旗杆 | 一段明确的 Impulse（推动浪），至少 3 根以上同向动量 K 线 |
| 旗面 | 平行通道（向推动方向的反方向倾斜），至少 5 根 K 线 |
| K 线特征 | 旗面内 K 线小实体、影线多、颜色混杂（回调特征） |
| 成交量（可见时） | 旗杆放量 + 旗面缩量 |

### 上升/下降楔形（Wedge）

| 特征 | 标准 |
|------|------|
| 结构 | 两条收敛的趋势线，价格在其中震荡 |
| 上升楔形 | 高点和低点都在上移，但高点上移速度 < 低点上移速度 -> 看跌中继/反转 |
| 下降楔形 | 高点和低点都在下移，但低点下移速度 < 高点下移速度 -> 看涨中继/反转 |
| 触碰次数 | 上下边界各至少被触碰 2 次（合计 >= 4 次） |
| 成交量（可见时） | 形态内部成交量逐渐递减 |

### 快速区分

| 判断 | Flag | Wedge |
|------|------|-------|
| 通道形状 | 平行 | 收敛 |
| 持续时间 | 较短（5~15 根 K 线） | 较长（10~30 根 K 线） |
| 可靠性 | 高（经典中继） | 中高（可能是中继也可能是反转） |

---

## 二、前提条件（缺一不可）

| 条件 | 标准 |
|------|------|
| 明确的前置推动 | 形态前有清晰的 Impulse（旗杆），不是从震荡中"长出来"的 |
| 趋势仍然有效 | 形态发生时趋势结构未被破坏（无 CHoCH） |
| HTF 同向 | 高周期趋势方向与形态突破方向一致 |
| 形态完整 | 至少 5 根 K 线形成旗面/楔形，不是 2 根就喊形态 |

> 任一不满足 -> 跳过此 Playbook。

---

## 三、突破确认

### 有效突破标准

**核心条件（必须全部满足）：**

| 条件 | 标准 |
|------|------|
| 方向正确 | 牛旗向上突破旗面上沿；熊旗向下突破旗面下沿 |
| 动量 K 线 | 突破 K 线实体 >= 旗面内平均 K 线实体的 2 倍 |
| 收盘越过 | 实体收盘在形态边界之外 |

**增强条件（可见时检查）：**

| 条件 | 标准 |
|------|------|
| 成交量 | 突破时成交量明显放大（>= 旗面内均量的 2 倍） |

### 突破失败信号

| 信号 | 行动 |
|------|------|
| 仅影线穿过形态边界 | 不是突破，继续等 |
| 突破后立即出现反向 K 线 | 可能是假突破，转 false-breakout-reversal |
| 向非预期方向突破（如牛旗向下突破） | 形态失效，趋势可能反转，放弃 |

---

## 四、入场规则

### 标准入场（推荐）

- **时机**：突破 K 线收盘确认后，下一根开盘入场
- **条件**：突破 K 线满足有效突破标准
- **止损**：形态内距离突破方向最远的极值点 + buffer

### 回踩入场（更保守）

- **时机**：突破后等价格回踩形态边界，出现支撑/阻力确认后入场
- **条件**：回踩 K 线温和（小实体、缩量），在形态边界附近出现拒绝 PA
- **止损**：回踩最低/高点 + buffer
- **风险**：可能不回踩直接走，错过入场

---

## 五、止损规则

| 入场类型 | 止损位置 | 说明 |
|---------|---------|------|
| 标准入场（做多） | 旗面/楔形最低点下方 + buffer | 如果跌破形态底部 = 形态失效 |
| 标准入场（做空） | 旗面/楔形最高点上方 + buffer | 同上 |
| 回踩入场 | 回踩极值点 + buffer | 更紧的止损 |

> buffer：首选 ATR x 0.3~0.5；无 ATR 时取形态内最大 K 线影线长度，或用止损位的 0.5~1% 作为固定偏移。

---

## 六、目标设定

### 等距投射法（主要方法）

```
做多：T1 = 突破点 + 旗杆长度
做空：T1 = 突破点 - 旗杆长度

旗杆长度 = 旗杆起点 到 旗杆终点 的价格距离
```

| 目标 | 位置 | 操作 |
|------|------|------|
| **T1** | 旗杆等距投射 | 到达后减仓 50%，止损移至成本价 |
| **T2** | 下一个关键阻力/支撑 | 剩余仓位跟踪止损 |

> 楔形的目标通常取形态最宽处的距离作为投射，而非"旗杆"。

---

## 七、不做清单（Disqualifiers）

| 条件 | 原因 |
|------|------|
| 没有明确的前置推动（旗杆） | 不是旗形，只是普通震荡 |
| 形态持续时间过长（> 30 根 K 线） | 形态能量耗散，突破动力不足 |
| 形态内部出现与推动等量级的反向 K 线 | 不是"喘气"，是反击 |
| HTF 方向与突破方向冲突 | 可能是末端楔形（反转信号），不做中继交易 |
| 趋势已走 3 波以上 + 楔形出现 | 末端楔形更可能是反转而非中继 |
| 重大事件即将公布 | 形态突破可能被事件否定 |

### 楔形的特殊注意

> 楔形在趋势末端出现时，更可能是**反转**信号而非中继。
> 判断方法：如果楔形前已有 3 次以上 BOS（趋势走了多波），优先视为反转楔形，不按此 Playbook 操作。

## 完整流程图

```
有明确的前置 Impulse（旗杆）？
  |
  +-- 否 -> 不是旗形/楔形
  |
  +-- 是 -> 形成了平行通道（旗形）或收敛结构（楔形）？
          |
          +-- 否 -> 等形态形成或不是此类形态
          |
          +-- 是 -> 形态向趋势方向突破？
                  |
                  +-- 否 -> 向反方向突破 = 形态失效，重新评估结构
                  |
                  +-- 是 -> 突破有效？（动量 K 线 + 收盘越过）
                          |
                          +-- 否 -> 可能是假突破，等待或转 false-breakout-reversal
                          |
                          +-- 是 -> 通过 pre-trade-checklist？
                                  |
                                  +-- 否 -> 被过滤掉
                                  |
                                  +-- 是 -> 入场
                                          -> 止损：形态极值外
                                          -> T1：旗杆等距投射
```

---

## AI 使用指引

- 用户问"这是旗形吗" -> 按形态标准逐项检查（前置推动 + 平行通道 + K 线特征）
- 用户问"楔形该怎么做" -> 先判断是中继楔形还是反转楔形（看趋势已走几波）
- 形态还在形成中 -> 告知"形态尚未完成，等突破确认"
- 突破后 -> 引导标准入场或回踩入场
- 向非预期方向突破 -> 明确说"形态失效，需要重新评估结构"
