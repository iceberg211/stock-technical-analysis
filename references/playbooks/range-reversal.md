# 区间边缘反转 (Range Reversal)

> **震荡不等于没机会，但机会只在边缘。区间中间是坟场。**
> 这个 Playbook 把"震荡行情"从模糊观望变成有限制、可执行的玩法。

---

## 一、成熟区间的定义

### 必须满足的标准

| 条件 | 标准 |
|------|------|
| 触碰次数 | 上沿和下沿各至少被触碰 **2 次**（合计 ≥ 4 次触碰） |
| 高低点走平 | Swing High 大致走平、Swing Low 大致走平，不形成 HH/HL 或 LH/LL |
| 时间长度 | 区间存在时间至少 **20 根 K 线**（当前时间框架） |
| 边界清晰 | 上下沿可以画出水平线/窄区域，而不是模糊的斜线 |

### 不是区间的情况（❌ 不套用此 Playbook）

| 情况 | 说明 |
|------|------|
| 刚震荡 2 根就喊区间 | 至少要看到 2 次上下沿触碰 |
| 区间中有趋势段 | 内部有明显的 BOS → 那是趋势中的盘整，不是区间 |
| 上下沿越来越窄 | 那是三角形收敛，不是区间 → 参考 `patterns/chart-patterns.md` |
| 波动剧烈无序 | 那是混乱市，不是区间 → 不做 |

---

## 二、可做区域 vs 禁区

```
                ┌─────────────────────┐  ← 区间上沿
                │   上沿 20% 区域     │  ← ✅ 可做空（如有拒绝信号）
                │─ ─ ─ ─ ─ ─ ─ ─ ─ ─│
                │                     │
                │   中间 60% 区域     │  ← ❌ 禁区，不做任何方向
                │                     │
                │─ ─ ─ ─ ─ ─ ─ ─ ─ ─│
                │   下沿 20% 区域     │  ← ✅ 可做多（如有拒绝信号）
                └─────────────────────┘  ← 区间下沿
```

> 🔑 **中间 60% = 无人区。** 在那里进场等于赌方向，没有结构优势。

### 20% 区域的计算

```
区间高度 = 上沿 - 下沿
上沿区域 = 上沿 往下 区间高度 × 20%
下沿区域 = 下沿 往上 区间高度 × 20%
```

## 三、触发条件（在边缘区域等待以下出现）

### 下沿做多触发

| 触发类型 | 具体条件 | 可靠性 |
|---------|---------|-------|
| **长下影线拒绝** | 在下沿区域出现长下影线（影线 ≥ 实体 2 倍） | ⭐⭐⭐ |
| **反转K线形态** | 锤子线 / 看涨吞没 / 启明星 | ⭐⭐⭐ |
| **多次触碰不破** | 价格第 3/4 次触碰下沿但无法跌破 | ⭐⭐⭐⭐ |
| **低级别反转结构** | 低时间框架出现 CHoCH + 多头 BOS | ⭐⭐⭐⭐ |
| **RSI 超卖反弹** | RSI < 30 且开始回升 | ⭐⭐（辅助） |

### 上沿做空触发（镜像）

| 触发类型 | 具体条件 |
|---------|---------|
| **长上影线拒绝** | 在上沿区域出现长上影线 |
| **反转K线形态** | 射击之星 / 看跌吞没 / 黄昏星 |
| **多次触碰不破** | 第 3/4 次触碰上沿无法突破 |
| **低级别反转结构** | 低时间框架 CHoCH + 空头 BOS |
| **RSI 超买回落** | RSI > 70 且开始回落 |

> ⚠️ **仅在边缘 20% 区域内出现的触发才有效。** 中间出现的反转形态一律忽略。

## 四、入场规则

### 标准入场

- **时机**：边缘触发K线收盘后，下一根开盘入场
- **方向**：下沿做多、上沿做空
- **仓位**：正常仓位（按 `risk/position-sizing.md` 计算）

### 保守入场

- **时机**：等低级别出现反转 BOS 后入场
- **仓位**：正常仓位
- **优势**：确认度更高，止损更紧

> ❗ **无论哪种入场，区间交易的仓位不应超过趋势交易的正常仓位。** 区间交易胜率可以但空间有限。

---

## 五、止损规则

| 方向 | 止损位置 | 说明 |
|------|---------|------|
| 下沿做多 | 区间下沿之外 + buffer | 如果跌破下沿 = 区间破位，方案失效 |
| 上沿做空 | 区间上沿之外 + buffer | 如果突破上沿 = 区间破位，方案失效 |

> 止损必须在区间之外。"放在区间内"= 被区间正常波动扫掉。
> buffer：首选 ATR × 0.5；无 ATR 时取区间内最大 K 线影线长度，或用止损位的 0.5–1% 作为固定偏移。

## 六、目标设定

| 目标 | 位置 | 操作 |
|------|------|------|
| **T1** | 区间中轴（区间高度 × 50%） | 到达后减仓 50%，止损移至成本价 |
| **T2** | 区间对侧（对面的沿） | 剩余仓位在接近对侧时考虑平仓 |

### 区间中轴计算

```
中轴 = (上沿 + 下沿) ÷ 2
```

> ⚠️ **区间交易的目标是有限的。** 不要期望区间交易能一波走趋势利润。T2 到对侧已经是最理想的情况。

---

## 七、区间失效 → 转突破

区间不会永远存在。当以下信号出现时，区间失效：

| 信号 | 含义 | 行动 |
|------|------|------|
| 动量K线突破上/下沿 | 区间可能结束 | 切换到 `breakout-retest.md` |
| 突破 + 回踩确认翻转 | 区间已确认结束 | 执行 breakout-retest 流程 |
| 区间内出现 BOS | 区间内开始形成趋势 | 重新评估市场状态 |

> 🔑 **如果你有区间做多的持仓，上沿被有效突破 = 好消息**（但仍按区间计划管理，不追加仓位）。

---

## 八、不做清单（Disqualifiers）

| 条件 | 原因 |
|------|------|
| 区间不满足成熟标准（< 4 次触碰） | 可能不是真正的区间 |
| 价格在区间中间 | 无结构优势 |
| 没有拒绝信号就想入场 | 在边缘 ≠ 必然反转 |
| HTF 有明确方向 + 当前在 HTF 趋势方向的区间沿 | 区间可能在趋势方向上被突破 |
| 区间越来越窄 | 是收敛三角形，不是区间 |
| 重大事件即将公布 | 区间交易依赖稳定环境 |

### HTF 偏向的特殊处理

> 如果 HTF 偏多 → 下沿做多的信心更高，上沿做空的信心降级
> 如果 HTF 偏空 → 上沿做空的信心更高，下沿做多的信心降级
> 这不影响入场规则，但影响仓位大小和目标预期。

## 完整流程图

```
市场状态 = 震荡区间？
  │
  ├── 否 → ❌ 不是区间反转单
  │
  └── 是 → 区间成熟？（≥4 次触碰 + 高低点走平 + ≥20 根K线）
          │
          ├── 否 → ⏳ 等区间成熟，或不是真正的区间
          │
          └── 是 → 价格在边缘 20% 区域？
                  │
                  ├── 否 → ❌ 在中间，不做
                  │
                  └── 是 → 出现拒绝/反转触发？
                          │
                          ├── 否 → ⏳ 在边缘等触发
                          │
                          └── 是 → 通过 pre-trade-checklist？
                                  │
                                  ├── 否 → ❌ 被过滤掉
                                  │
                                  └── 是 → ✅ 入场
                                          → T1 = 中轴，T2 = 对侧
                                          → 止损在区间外
```

## AI 使用指引

- 用户问"这是震荡吗" → 按成熟区间标准逐项检查
- 用户问"震荡中怎么做" → 明确指出可做区域（边缘 20%）和禁区（中间 60%）
- 价格在中间 → 明确说"目前在区间中间，不建议入场，等靠近边缘"
- 价格在边缘但无触发 → 说"在边缘但缺少拒绝信号，继续等"
- 区间被突破 → 主动提示"区间可能结束，切换到突破回踩逻辑"
