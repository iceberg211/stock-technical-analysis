# 顺势回调 (Trend Pullback)

> **在健康趋势中，等回调到位、等触发出现、顺势入场。**
> 这是胜率最高的 setup 之一：趋势提供方向，回调提供价格优势。

---

## 一、前提条件（缺一不可）

| 条件 | 标准 | 引用 |
|------|------|------|
| 趋势清晰 | HH+HL（做多）或 LL+LH（做空），至少 2 个同方向 BOS | `core/market-structure.md` |
| 推动浪健康 | Impulse 实体大、方向一致、速度快 | `core/market-structure.md` 第四节 |
| 没有 CHoCH | 最近不能出现反向的 CHoCH | `core/market-structure.md` 第六节 |
| HTF 同向 | 高周期趋势方向与本级别一致（多图时必查） | `workflows/chart-analysis-workflow.md` Step 1.5 |

> ❌ 任何一条不满足 → **这不是顺势回调单，跳过此 Playbook。**

---

## 二、回调质量评估

回调质量条件分为两层：

| 层级 | 说明 | 作用 |
|------|------|------|
| **核心条件** | 仅凭 K线和结构可判断，截图即可 | 必须满足，否则不值得等触发 |
| **增强条件** | 需要指标工具（Fib/Volume/ATR） | 可见时加分，不可见不扣分 |

### 健康回调（✅ 可以等触发）

**核心条件（必须全部满足）：**

| 特征 | 说明 |
|------|------|
| K线特征 | 小实体、影线多、颜色混杂 |
| 结构保持 | 未破坏前一个 Major Swing（做多时未跌破前 HL） |
| 回调速度 | 比推动浪慢（时间花得更多） |
| 无反向动量 | 回调中未出现与推动浪等量级的反向 K 线 |

**增强条件（可见时检查，提升信心）：**

| 特征 | 说明 |
|------|------|
| 回调深度 | Fib 0.382 – 0.618 区间内 |
| 成交量 | 递减或低于推动浪 |

### 不健康回调（❌ 不做）

**核心否决（任一出现即放弃）：**

| 特征 | 说明 |
|------|------|
| 回调过猛 | 出现反向动量 K 线（实体 ≥ 推动浪 K 线） |
| CHoCH 出现 | 回调破坏了趋势内部结构 |
| 跌破关键位 | 破坏了趋势的 Strong Level |

**增强否决（可见时检查）：**

| 特征 | 说明 |
|------|------|
| 回调过深 | 超过 Fib 0.786 |
| 放量下杀 | 回调中成交量不减反增 |

> 🔑 **核心判断**：健康回调像"喘气"，不健康回调像"反击"。

## 三、触发信号（在回调支撑区等待以下任一出现）

### 做多回调触发

| 触发类型 | 具体条件 | 可靠性 |
|---------|---------|-------|
| **反转 K 线** | 在 Fib 0.382–0.618 区域出现锤子线 / 吞没 / 启明星 | ⭐⭐⭐ |
| **低级别 BOS** | 切换到更低时间框架，看到反向 CHoCH + 同向 BOS | ⭐⭐⭐⭐ |
| **支撑拒绝** | 触碰前 HL 区域 + 长下影线 + 缩量 | ⭐⭐⭐ |
| **指标确认** | RSI 从超卖区回升 / MACD 柱缩小并开始反转 | ⭐⭐（辅助，不单独使用） |

### 做空回调触发（镜像）

| 触发类型 | 具体条件 |
|---------|---------|
| **反转 K 线** | 在 Fib 0.382–0.618 区域出现射击之星 / 吞没 / 黄昏星 |
| **低级别 BOS** | 低时间框架出现反向 CHoCH + 空头 BOS |
| **阻力拒绝** | 触碰前 LH 区域 + 长上影线 |

> ⚠️ **没有触发就不入场**。回调到位 ≠ 直接进，必须等价格"告诉你"方向已经恢复。

---

## 四、入场规则

### 激进入场

- **时机**：反转 K 线收盘后，下一根开盘入场
- **适合**：强趋势 + 明显支撑区 + 明确反转形态
- **风险**：可能被进一步回调扫止损

### 保守入场

- **时机**：等低级别出现同向 BOS 确认后入场
- **适合**：趋势不够强 / 回调偏深 / 需要更多确认
- **风险**：入场价格劣势，但止损更紧

---

## 五、止损规则

| 类型 | 止损位置 | 说明 |
|------|---------|------|
| 做多 | 回调最低点下方 + buffer | 必须在趋势 Strong Level 之外 |
| 做空 | 回调最高点上方 + buffer | 同上 |

### buffer 说明

- buffer 是为了避免被正常波动的影线扫到
- **首选**：用 ATR（平均真实波动范围）的 0.3–0.5 倍作为 buffer
- **无 ATR 时的替代方案**：取回调段内最大 K 线影线长度作为 buffer，或使用止损位的 0.5–1% 作为固定偏移
- 如果止损距离太大导致 R:R < 1.5:1 → **放弃此笔交易**

---

## 六、目标设定

| 目标 | 位置 | 操作 |
|------|------|------|
| **T1** | 前高/前低（趋势方向的最近 Swing 极值） | 到达后减仓 50%，止损移至成本价 |
| **T2** | 等距运动（Measured Move）或下一个关键阻力/支撑 | 剩余仓位跟踪止损 |
| **T3**（可选） | 更高级别的阻力/支撑 | 仅在强趋势中考虑 |

> 入场后的详细管理规则 → 见 `checklists/in-trade-management.md`

---

## 七、不做清单（Disqualifiers）

以下任一成立 → **跳过此机会，等下一个**：

| 条件 | 原因 |
|------|------|
| 趋势已走了 3 波以上 | 趋势末端，回调入场胜率骤降 |
| 回调超过 0.786 | 已不是"回调"，是可能反转 |
| HTF 方向冲突 | 逆大势做小势，风险不对称 |
| 重大事件即将公布 | 数据/财报可能使技术分析失效 |
| 同品种已有同向持仓 | 避免重复暴露，见 `risk/position-sizing.md` |
| 前一次同类 setup 刚被止损 | 连续同类失败，等市场重新证明结构 |

## 完整流程图

```
趋势清晰（≥2 BOS + 无 CHoCH）？
  │
  ├── 否 → ❌ 不是顺势回调单
  │
  └── 是 → 正在回调？
          │
          ├── 否 → ⏳ 等回调
          │
          └── 是 → 回调健康？（Fib 0.382–0.618 + 缩量 + 小K线）
                  │
                  ├── 否 → ❌ 回调质量差，放弃
                  │
                  └── 是 → 出现触发？（反转K线 / 低级别BOS / 支撑拒绝）
                          │
                          ├── 否 → ⏳ 继续等
                          │
                          └── 是 → 通过 pre-trade-checklist？
                                  │
                                  ├── 否 → ❌ 被过滤掉
                                  │
                                  └── 是 → ✅ 执行入场
                                          → 按 position-sizing 定仓位
                                          → 按 in-trade-management 管持仓
```

## AI 使用指引

- 用户问"可以回调做多/做空吗" → 按上方流程逐项检查
- 回调到位但无触发 → 明确说"回调到位但缺少触发，继续等"
- 回调不健康 → 明确说"回调质量差"并指出具体原因
- 所有结论必须对应上方表格中的某一条规则，不凭感觉判断
- 给出交易方案时，引用本文件中的止损/目标规则
