# 突破回踩 (Breakout Retest)

> **突破不是入场信号，回踩确认才是。大多数突破失败的原因：太早追、没等回踩。**
> 这个 Playbook 的核心是"等"——等突破、等回踩、等确认。

---

## 一、有效突破的定义

突破有效性条件分为两层：

**核心条件（必须全部满足）：**

| 条件 | 标准 | 引用 |
|------|------|------|
| 动量K线 | 突破K线是 Momentum Candle（实体 ≥ 前 3 根的 2 倍） | `core/price-action.md` 第三节 |
| 收盘越过 | K线实体收盘在关键位**之外**，不仅仅是影线穿过 | `core/price-action.md` 第五节 |
| 关键水平位 | 被突破的是真正的关键支撑/阻力，不是随便画的线 | `core/support-resistance.md` |

**增强条件（可见时检查，提升信心）：**

| 条件 | 标准 | 引用 |
|------|------|------|
| 成交量 | 突破时成交量明显放大 | `core/price-action.md` 第六节 |

> ❌ 核心条件任一不满足 → **不是有效突破，不进入回踩等待。**
> 增强条件不满足不否决，但信心等级下降。

---

## 二、假突破识别（任一出现 → 突破无效）

| 信号 | 说明 |
|------|------|
| 仅影线穿过 | K线影线越过关键位但实体收回，没有真正站上/跌破 |
| 无量突破 | 突破时成交量没有明显放大 |
| 收盘失败 | 突破K线收盘回到关键位内侧 |
| 立即反向 | 突破后下一根K线立即出现反向动量K线 |
| 连续小K线试探 | 多根小K线反复试探，没有一根有力度的突破 |

> 🔑 **假突破 = 最好做反向的机会之一**（参见 `core/market-structure.md` Model B: Failure Test）

---

## 三、回踩等待规则

### 回踩的本质

> 原来的阻力 → 突破后变支撑（做多）
> 原来的支撑 → 跌破后变阻力（做空）
> 回踩就是测试这个"翻转"是否成立。

### 回踩接单条件

**核心条件（必须满足）：**

| 条件 | 标准 |
|------|------|
| 价格回到突破位附近 | 回踩到原关键位附近 |
| 回踩K线温和 | 小实体、影线多，不是动量K线反砸 |
| 出现拒绝PA | 在回踩位出现反转K线 / 长影线 / 低级别反转结构 |

**增强条件（可见时检查）：**

| 条件 | 标准 |
|------|------|
| 回踩缩量 | 回踩过程中成交量递减 |
| 像差精度 | 回踩到原关键位 ± ATR × 0.5 以内 |

### 回踩失败判定

| 信号 | 行动 |
|------|------|
| 回踩跌破突破起点 | ❌ 方案彻底失效，不入场 |
| 回踩带动量K线反穿 | ❌ 突破已被否定，可能是假突破 |
| 回踩后横盘过久（>20 根K线） | ⚠️ 信号衰减，降级处理或放弃 |

---

## 四、不回踩怎么办？

> **默认：不追。**

### 唯一例外（必须同时满足）

| 条件 | 标准 |
|------|------|
| 极强趋势 | HTF 趋势清晰 + 当前突破是 BOS 延续 |
| 连续动量 | 突破后连续 2 根以上同向动量K线 |
| 小级别回调 | 在更低时间框架中出现浅回调（Fib 0.236–0.382） |
| 仓位缩减 | 仓位降至正常的 50%（因为没有理想止损位） |

> ⚠️ 即使满足以上条件，这仍然是一笔"非标准"交易，标记为激进入场。

---

## 五、入场规则

### 标准入场（推荐）

- **时机**：回踩到位 + 出现拒绝 PA → 拒绝K线收盘后入场
- **止损**：回踩最低点（做多）/ 最高点（做空）之外 + buffer

### 激进入场

- **时机**：回踩刚触碰突破位即入场，不等确认
- **止损**：突破起点之外
- **注意**：仓位必须缩减（见 `risk/position-sizing.md`）

---

## 六、止损规则

| 类型 | 止损位置 | 说明 |
|------|---------|------|
| 做多回踩 | 回踩最低点下方 + buffer | 或突破起涨点下方，取更近的 |
| 做空回踩 | 回踩最高点上方 + buffer | 或跌破起跌点上方，取更近的 |
| 不追单 | 突破K线实体中点 | 仅适用于"不回踩例外"情况 |

> buffer：首选 ATR × 0.3–0.5；无 ATR 时取回踩段内最大 K 线影线长度，或用止损位的 0.5–1% 作为固定偏移

---

## 七、目标设定

| 目标 | 位置 | 操作 |
|------|------|------|
| **T1** | 突破幅度等距投射（Measured Move） | 到达后减仓 50%，止损移至成本价 |
| **T2** | 下一个关键阻力/支撑 | 剩余仓位跟踪止损 |

### 等距投射计算

```
做多：T1 = 突破位 + (突破位 - 前一个 Swing Low)
做空：T1 = 突破位 - (前一个 Swing High - 突破位)
```

---

## 八、不做清单（Disqualifiers）

| 条件 | 原因 |
|------|------|
| 突破不满足 4 条有效性标准 | 可能是假突破 |
| 回踩跌破/突破起点 | 翻转失败 |
| HTF 方向与突破方向冲突 | 逆大势突破，失败率高 |
| 区间已被突破过多次（来回穿） | 不是真正的突破，是噪音 |
| 重大事件即将公布 | 事件可能否定技术信号 |
| 同时匹配了 trend-pullback | 优先按 trend-pullback 处理（更高胜率） |

## 完整流程图

```
关键水平位被突破？
  │
  ├── 否 → ❌ 不是突破回踩单
  │
  └── 是 → 突破有效？（动量K线 + 收盘越过 + 放量 + 关键位）
          │
          ├── 否 → ⚠️ 可能是假突破，考虑反向（Model B）
          │
          └── 是 → 等回踩
                  │
                  ├── 回踩到位 + 出现拒绝PA？
                  │       │
                  │       ├── 是 → 通过 pre-trade-checklist？
                  │       │       │
                  │       │       ├── 是 → ✅ 标准入场
                  │       │       └── 否 → ❌ 被过滤掉
                  │       │
                  │       └── 否 → 回踩失败？（破起点 / 动量反穿）
                  │               │
                  │               ├── 是 → ❌ 方案失效
                  │               └── 否 → ⏳ 继续等
                  │
                  └── 一直不回踩？
                          │
                          ├── 满足"不回踩例外"4 条？ → ⚠️ 激进入场（半仓）
                          └── 否 → ❌ 放弃，等下一个机会
```

---

## AI 使用指引

- 用户问"这是不是突破" → 严格对照 4 条有效突破标准
- 用户问"可以追吗" → 默认回答"等回踩"，仅在极端条件下建议追
- 用户问"回踩到位了吗" → 检查回踩接单的 4 个条件
- 看到突破后立即回落 → 主动提示"可能是假突破"
- 给出交易方案时，必须明确标注是"标准入场"还是"激进入场"
